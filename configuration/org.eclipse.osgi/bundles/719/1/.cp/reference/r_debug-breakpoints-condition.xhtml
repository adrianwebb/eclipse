Condition / Expression Property of R Breakpoints

Each breakpoint can have a unique condition that determines if the will be suspended at the breakpoint position. A conditional breakpoint has a question mark overlay on its icon.

A condition for a breakpoint can be any valid R expression. The expression is evaluated in the scope (R frame) of the breakpoint location. If it evaluates to an atomic R logical value TRUE, the thread is suspended as usual. Otherwise it is skipped like a disabled breakpoint.

Changing the Settings

A condition can be set, enabled/disabled and removed in the Breakpoint view.

  1. Select the breakpoint to change.
    The Breakpoint view shows properties supported by the selected breakpoint in the detail pane below or beside the breakpoints list.
  2. The checkbox Conditional / Expression allows to enable and disable the condition.
  3. If the checkbox is enabled, the user can enter the expression of the condition in the text field below. The common Save command (File > Save) applies the changed expression to the breakpoint; it is also automatically saved when selecting another breakpoint.
