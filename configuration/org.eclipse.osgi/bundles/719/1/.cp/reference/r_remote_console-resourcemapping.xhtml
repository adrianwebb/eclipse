Remote Folders / Folder Mapping

Idea

If the same file system directory is accessible locally (on the client) and on the server, folder mapping allows to make use of this match. The folder mapping specifies the translation of both path.

Remote Folders are configured in the preference page StatET > Run/Debug > Remote Folders.

Example

For example the working data are store in a directory /home/stephan/mydata on the server with the IP 192.168.10.19. This directory is mounted on the Windows client to L:\mydata. So you create a folder mapping entry:

Detail

The Remote Host can also be specified by the host name, but the name must be resolvable to an IP address.

It is irrelevant where the real directory is stored. It can be a client directory, server directory or on another third storage system. A folder mapping applies to the directory itself, all subdirectories and all containing files.

How It Works

The mapping works completely transparently for the user.

For example when sourcing a file (e.g. L:\mydata\project1\script.R), usually StatET knows only the local file path, but R on the server needs the path in the file system of the server. If the file is inside a directory with a configured folder mapping, StatET can translate automatically the correct path (to /home/stephan/mydata/project1/script.R).

Note: not yet all functions of the R console supports path mapping.
